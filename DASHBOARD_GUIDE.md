# 관리자 대시보드 사용 가이드

## 접속 방법

1. 개발 서버 실행:
   ```bash
   npm run dev
   ```

2. 브라우저에서 접속:
   ```
   http://localhost:3000/dashboard
   ```

## 사용 방법

### 1. 세션 코드 입력
- Supabase에서 생성한 세션의 `session_code`를 입력합니다
- 예: `test-session-001`

### 2. 관리자 키 입력
- `.env.local` 파일에 설정한 `ADMIN_KEY` 값을 입력합니다
- 예: `lgchem-leadership-admin-2026-secure`

### 3. 데이터 조회
- "데이터 조회" 버튼을 클릭합니다
- API를 통해 집계 데이터를 가져옵니다

## 표시되는 데이터

### 1. 세션 요약
- **총 응답 수**: 해당 세션의 총 제출 응답 수
- **유형 종류**: 발견된 리더십 유형의 개수
- **데이터 축**: 항상 4개 (동기, 유연성, 방향성, 소통)

### 2. 리더십 유형 분포
- 각 유형별 응답자 수와 비율
- 진행 바를 통한 시각적 표현
- 내림차순 정렬 (가장 많은 유형부터)

### 3. 4축 통계
각 축별로 다음 정보를 표시합니다:
- **평균 점수**: 해당 축의 평균 점수
- **표준편차**: 점수의 분산 정도
- **극성 분포**: 각 극성(intrinsic/extrinsic, change/system 등)의 비율

#### 축별 정보
- **동기 (Motivation)**: 내재적 / 외재적 / 균형
- **유연성 (Flexibility)**: 변화 / 관리 / 균형
- **방향성 (Direction)**: 성과 / 관계 / 균형
- **소통 (Communication)**: 지시 / 참여 / 균형

### 4. 디브리핑 인사이트
집계 데이터를 바탕으로 자동 계산되는 인사이트:

- **편차가 큰 축 (TOP 2)**
  - 표준편차가 높은 축들
  - 참가자 간 점수 분산이 큰 영역

- **한쪽으로 치우친 축 (TOP 2)**
  - 특정 극성으로 집중된 축들
  - 참가자들이 비슷한 성향을 보이는 영역

## 디자인 특징

- **Glass morphism 스타일**: 반투명 카드 디자인
- **반응형 레이아웃**: 모바일, 태블릿, 데스크톱 지원
- **브랜드 컬러**: Purple, Magenta, Light Blue 그라디언트
- **시각적 진행 바**: 데이터 비율을 직관적으로 표현

## 문제 해결

### "세션을 찾을 수 없습니다" 에러
- 세션 코드가 올바른지 확인하세요
- Supabase의 `sessions` 테이블에서 세션이 생성되었는지 확인하세요

### "유효하지 않은 관리자 키입니다" 에러
- `.env.local` 파일의 `ADMIN_KEY` 값을 확인하세요
- 입력한 관리자 키가 환경 변수와 일치하는지 확인하세요

### 데이터가 표시되지 않음
- 해당 세션에 제출된 응답이 있는지 확인하세요
- Supabase의 `responses` 테이블에서 데이터를 확인하세요

### 빈 화면 또는 로딩 상태
- 브라우저 콘솔에서 에러 메시지를 확인하세요
- 서버 로그를 확인하세요 (`npm run dev` 실행 터미널)
